# MeetingMind 실측용 회의 스크립트 (2인·단축판)

> **목적**: 짧은 회의 1건으로 핵심 기능 전부 트리거
> **길이**: 약 8~10분 (녹음 기준)
> **발화 수**: 약 35개
> **화자**: 2명 (1인 2역 가능)
> **권장 환경 변수**: `MM_TIME_OVER_ALERT_MIN=2.0` (짧은 회의에서 time_over 검증)

---

## 📋 기능별 기대 발동 맵

| 기능 | 발동 방식 | 발동 위치 |
|---|---|---|
| 화자 분리 | 2명 목소리 구분 | 전체 |
| 토픽 감지 경로 A | 1차 미매칭 (대부분) | 본문 발화 |
| 토픽 감지 경로 B | "다음 안건"+"넘어가서" 동시 | 안건 2 시작 |
| 토픽 감지 경로 C | 1차 통과+2차 미통과 → LLM | 안건 3 시작 |
| 토픽 강제 LLM 판정 | 12발화 누적 시 | 자연 발생 |
| 쟁점 구조화 | 500토큰 누적 배치 | 안건당 1회 |
| 자료 수집 | 쟁점 구조 갱신 시 1회 | 안건 1·2 |
| consensus | "동의합니다"/"좋습니다" 단독 (10자 이내) | 안건 1·3 끝 |
| info_needed | "확인해봐야"/"데이터를 봐야" | 안건 1 |
| no_decision | 이전 안건 decision 없이 전환 | 안건 3 시작 |
| loop | 같은 단어 반복 | 안건 2 |
| silence | 5초+ 무음 | 안건 3 |
| time_over | 안건 시간 초과 | 안건 2 |

---

## 🎤 녹음 가이드

- **화자 2명**:
  - **김팀장 (A)** — 진행자. 중저음, 차분
  - **이대리 (B)** — 팀원. 빠르고 명확
- **트리거 키워드는 한 글자도 바꾸지 말 것** ("좋네요" X, "좋습니다" O)
- **consensus 발화는 짧게, 단독 문장으로** (10자 초과 시 필터링)
- **[5초 침묵] 지시는 정확히 지킬 것**

## 🗓️ 팀 정기 회의

**김팀장**: 안녕하세요, 오늘 세 가지만 짧게 정하겠습니다. 회식 예산, 사무실 자리 배치, 워크숍 일정 순으로요.

**이대리**: 네, 시작하시죠.

## 안건 1: 다음달 회식 예산 (목표 2분)

### [00:20] 의견 충돌

**김팀장**: 첫째, 회식 예산을 1인당 얼마로 잡을지부터요. 의견 주시죠.

**이대리**: 저는 1인당 5만 원으로 올리는 데 찬성입니다. 작년보다 식당 가격이 많이 올랐어요.

**김팀장**: 저는 신중합니다. 작년에도 올렸는데 만족도가 그만큼 안 올랐어요.

<!-- 쟁점 구조: positions=2 → 자료 수집 트리거 -->

**이대리**: 작년이랑은 식당 가격대가 또 달라요.

**김팀장**: 그 부분은 정확한 수치를 확인해봐야 합니다.

<!-- 트리거: info_needed -->

**이대리**: 지난번 사내 설문 보면 음식이 부실해서 아쉬웠다는 응답이 60퍼센트였어요.

**김팀장**: 그 데이터를 봐야 판단할 수 있어요.

<!-- 트리거: info_needed -->

### [01:30] 합의

**이대리**: 그럼 4만 5천 원 선에서 잡고 분기마다 다시 확인하는 걸로요.

**김팀장**: 좋습니다.

<!-- 트리거: consensus -->

**이대리**: 동의합니다.

<!-- 트리거: consensus (중복 방지 시 1회만) -->

## 안건 2: 사무실 자리 배치 (목표 3분) — 경로 B

### [02:30] 토픽 전환 — 2차 필터 통과 (LLM 0회)

**김팀장**: 네, 다음 안건으로 넘어가서 사무실 자리 배치 얘기 해보죠.

<!-- 트리거: 토픽 감지 경로 B -->

**이대리**: 창가 자리를 팀별로 돌아가며 쓰는 안을 생각해 봤어요.

**김팀장**: 창가 자리는 업무 집중도가 사람마다 달라서, 그냥 두는 게 나을 수도 있어요.

### [03:00] loop 유도 — "창가" 반복

**이대리**: 창가 안 쓰던 분들이 계속 불만이세요.

**김팀장**: 창가 자리 옮기려면 짐 정리만 한참이에요.

**이대리**: 창가 배치는 분기에 한 번 정도면 부담 없을 거예요.

**김팀장**: 그래도 창가 자리 자체가 부족한 게 진짜 문제죠.

<!-- 트리거: loop ("창가" 4회+ 반복) -->

### [04:00] 결론 보류 — decision 비움

**이대리**: 우선 사내 설문부터 돌리고 다음 회의에서 정하시죠.

**김팀장**: 네 그렇게 하시죠. 일단 보류로 두고요.

<!-- "그렇게 하시죠"는 consensus 키워드 아님 → decision null 유지 -->

### [05:00] time_over 진입

**김팀장**: 이 안건 시간이 좀 됐네요.

<!-- 트리거: time_over -->

**이대리**: 다음 회의 때 설문 결과 들고 다시 보겠습니다.

## 안건 3: 워크숍 일정 (목표 2분) — 경로 C

### [05:30] 토픽 전환 — 1차 통과·2차 미통과 → LLM fallback

**김팀장**: 자 이제 마지막 워크숍 얘기 해보죠.

<!-- 트리거: 토픽 감지 경로 C -->
<!-- 동시 트리거: no_decision (안건 2가 decision 없이 종결) -->

**이대리**: 다음 달 셋째 주 금토로 1박 2일 가는 안이에요.

**김팀장**: 1박은 좀 부담스러워요. 당일치기로 가는 게 나아요.

### [06:30] silence 트리거

**이대리**: 음...

**[5초 침묵 — 정확히 5초 이상 무음]**

<!-- 트리거: silence -->

**이대리**: 그럼 토요일 당일치기로 가되, 오전 일찍 시작해서 저녁까지 길게 쓰는 건 어때요.

**김팀장**: 오전 시작이면 일정 짜기 괜찮겠네요.

### [07:30] 합의

**이대리**: 장소는 다음 주까지 확정하는 조건이면 됩니다.

**김팀장**: 좋습니다.

<!-- 트리거: consensus -->

**이대리**: 동의합니다.

### [08:00] 마무리

**김팀장**: 오늘 정리하면, 회식은 4만 5천 원, 자리 배치는 설문 후 재논의, 워크숍은 토요일 당일치기. 마치겠습니다.

**이대리**: 네 감사합니다.

## 🧪 실측 검증 체크리스트

### 1. 화자 분리
- [ ] speaker_id 2종 분리
- [ ] 화자별 발화 수 ≈ 김팀장 18 / 이대리 17

### 2. 토픽 감지 (3경로)
- [ ] 경로 B 발동: 안건 2 시작 — LLM 0회
- [ ] 경로 C 발동: 안건 3 시작 — LLM 1회
- [ ] 경로 A: 본문 발화 — LLM 0회
- [ ] 강제 LLM 판정 (12발화 누적): 발동 여부 확인

### 3. 쟁점 구조화
- [ ] 안건당 1회 배치 LLM 호출
- [ ] Position 정확도: 안건 1 [이대리=인상 / 김팀장=신중], 안건 2 [이대리=순환 / 김팀장=유지], 안건 3 [이대리=1박2일 / 김팀장=당일치기]

### 4. 자료 수집 (쟁점 구조 갱신 트리거)
- [ ] 안건 1 쟁점 갱신 시점에 검색 1회
- [ ] 안건 2 쟁점 갱신 시점에 검색 1회
- [ ] 발화당 검색은 발생 안 함

### 5. 개입 트리거 6종

## ⚠️ 자주 하는 실수

1. "좋습니다" → "좋네요/좋아요" 변형 시 consensus 미발동
2. consensus 키워드 뒤에 긴 설명 붙이면 10자 초과로 필터링
3. 5초 침묵 중 "음..." 소리 냄 → silence 미발동
4. "창가" → "창문/창쪽" 변형 시 loop 미발동
5. 토픽 전환 키워드 ("다음 안건으로 넘어가서", "자 이제") 임의 변경 시 경로 B/C 테스트 실패
