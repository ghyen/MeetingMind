# MeetingMind 실측용 회의 스크립트

> **목적**: 실측 데이터 1장 확보 (경로 A/B/C 분포, 트리거 발동 횟수, LLM 실제 호출 횟수)
> **길이**: 약 25분 (녹음 기준)
> **발화 수**: 약 135개 (짧은 발화 30% + 중간 발화 50% + 긴 발화 20%)
> **커버리지**: 프로젝트의 **모든 기능**을 한 회의로 트리거

---

## 📋 스크립트 설계 맵 (기능별 기대 발동)

| 기능 | 발동 방식 | 발동 위치 |
|-----|---------|---------|
| **화자 분리** | 4명 목소리 구분 | 전체 |
| **토픽 감지 경로 A** (현 토픽 유지) | 1차 키워드 미매칭 | 발화 대부분 |
| **토픽 감지 경로 B** (2차 확정, LLM 0회) | "다음 안건"+"넘어가서" 등 2개+ 동시 | 안건 2 시작 |
| **토픽 감지 경로 C** (LLM fallback) | 1차만 통과, 2차 미통과 | 안건 3 시작 |
| **쟁점 구조화** | 500토큰 누적 시 배치 LLM | 안건당 2-3회 |
| **자료 수집** | `positions >= 2 && consensus == null` + 엔티티 | 안건 1, 안건 2 |
| **트리거 1 - consensus** | "좋습니다"/"동의합니다" 단독 (10자 초과 안됨) | 안건 1 끝, 안건 3 끝 |
| **트리거 2 - info_needed** | "확인해봐야"/"데이터를 봐야"/"자료가 있나"/"찾아봐야" | 안건 1, 안건 2 |
| **트리거 3 - no_decision** | 이전 안건 decision 없는 채 전환 | 안건 3 시작 |
| **트리거 4 - loop** | 같은 단어 3회+ 반복 (2자 이상, 불용어 제외) | 안건 2 중반 |
| **트리거 5 - silence** | 5초+ 무음 | 안건 3 |
| **트리거 6 - time_over** | 안건 10분+ 경과 | 안건 2 |

## 🎤 녹음 가이드

1. **환경**: 조용한 회의실 또는 1인 스튜디오 (1인 4역 연기 시 목소리 구분 명확히)
2. **화자 4명**:
   - **김팀장 (A)** — 마케팅팀장, 진행자. 중저음, 차분한 톤
   - **이리드 (B)** — 퍼포먼스 마케팅 리드. 빠르고 자신감 있는 톤
   - **박재무 (C)** — 재무담당. 신중하고 또박또박, 숫자 강조
   - **최기획 (D)** — 기획자. 부드럽고 설명조
3. **페이싱**: 발화 사이 1-2초 자연스러운 간격. 명시된 **[5초 침묵]** 지시는 반드시 지킬 것.
4. **트리거 키워드는 한 글자도 안 바꿀 것.** "좋습니다"를 "좋네요"로 바꾸면 consensus 발동 안 함.
5. **consensus 발화 주의**: "좋습니다"/"동의합니다" 뒤에 긴 부가 설명 붙이면 발동 안 됨 (10자 초과 시 필터링). **단독으로 말하거나, 짧은 마침표 문장으로 끝낼 것**.
